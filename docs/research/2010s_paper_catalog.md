# 2010年代論文カタログ

## 目的

`investor2` の検証対象を、2010年代の実証アセット・プライシング、ファクター投資、アノマリー再現、検証方法へ段階的に広げるための一次情報カタログ。論文の存在確認と、投資シグナルの再現確認は分離する。

## 現在の収集範囲

24件を2010–2019年の各年に配置した。シグナル論文だけでなく、ファクターモデル比較、取引コスト、出版後減衰、多重検定を含める。これにより「高い論文数」ではなく「検証ループを閉じられる論文集合」を作る。

| 年 | 件数 | 主なテーマ |
|---|---:|---|
| 2010 | 1 | 情報拡散・業種間予測 |
| 2011 | 3 | 注目、宝くじ型株、国際ファクター |
| 2012 | 1 | 時系列モメンタム |
| 2013 | 4 | 収益性、価値・モメンタム、投資ショック、財務困難 |
| 2014 | 1 | 低ベータ |
| 2015 | 3 | q-factor、FF5、裁定非対称性 |
| 2016 | 5 | 多重検定、5因子、出版後減衰、モメンタム暴落、取引コスト |
| 2017 | 3 | ボラティリティ管理、特性選択、ミスプライシング因子 |
| 2018 | 2 | 因子選択、ベイズ型モデル比較 |
| 2019 | 1 | Quality Minus Junk |

## 検証への投入順

1. 既存のFF5データでFama–French (2015)を再実行する。
2. 公開データで再構築しやすい、低ベータ・収益性・モメンタム・ボラティリティ管理を優先する。
3. CRSP/Compustat、Google SVI、先物、国際株式データが必要な論文は、データ出所とライセンスを先に固定する。
4. Harvey–Liu–Zhu、McLean–Pontiff、Novy–Marx–Velikov、Green–Hand–Zhangを、すべての個別シグナルの判定基準として使う。

## 状態の意味

- `metadata_collected`: 一次情報で論文メタデータと検証仮説を確認済み。再現結果ではない。
- `registry_existing`: `paper_factor_registry.json` に検証対象が存在する。
- `data_pending`: 将来、データと固定OOS期間が揃ったらレジストリへ追加する状態。

機械可読な正本は [2010s_paper_catalog.json](./2010s_paper_catalog.json)。スキーマ検査は `uv run --python 3.12 python scripts/validate_paper_catalog.py` で実行する。
