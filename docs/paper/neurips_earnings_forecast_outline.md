# AAARTS: 自律的エージェント型アルファ取引システムによる財務利益予測

**対象会議:** NeurIPS 2026（Datasets and Benchmarks Track または Applied AI）  
**焦点:** 日本語財務諸表（EDINET）を用いた利益予測

---

## 論文のストーリー展開

### 1. 導入
*   課題: 企業の業績予測は、クオンツ投資における最も重要なタスクの一つである。しかし、株価や純利益の数値といった従来の定量データのみを用いる予測モデルは、市場の非効率性を十分に捉えられず、限界に直面している。
*   解決策: 財務諸表の定性的データ（MD&A、リスク要因、事業状況等のテキスト）には、次期業績を予見する「暗黙のシグナル」が潜んでいる。
*   提案手法: 本論文では、LLM を用いて定性的財務テキストから自律的にアルファ（予測シグナル）を発見・検証するシステム AAARTS（Autonomous Agentic Alpha Trade System）を提案する。
*   貢献:
    1.  テキスト情報に基づく業績予想のエンドツーエンドのパイプラインの構築。
    2.  日本語の複雑な財務文書（EDINET）を対象とした、Sakana AI の `EDINET-Bench` を用いた厳密な評価。
    3.  AIの推論根拠（Reasoning）の分析による、業績変動メカニズムの解明。

### 2. 関連研究

#### 2.1 金融言語モデリングと金融分野における自然言語処理
定量ファイナンスにおける自然言語処理の初期応用は、辞書ベースのアプローチや表層的な機械学習モデルに主に依存し、企業開示情報や金融ニュースのセンチメント抽出を目的としていた。トランスフォーマー型アーキテクチャの導入により、特定ドメイン向けの言語モデルの開発が進み、FinBERT などが金融コーパスに対する事前学習を通じて感情分類や固有表現抽出で顕著な改善を達成した。近年では大規模言語モデル（LLM）が複雑なタスクにおける推論能力を示している。BloombergGPT など、金融文書と一般ウェブ文書の大規模ハイブリッドデータセットから学習されたモデルは、金融用語や意味関係の表現力を高めた。一方で、現状のモデルは、テキスト分類やセマンティック検索、あるいは一般的な要約には適しているものの、構造化された定量予測や複数財務諸表に跨る予測には最適化されていない。

#### 2.2 定量取引におけるエージェント型LLMループ
金融市場における受動的なテキスト処理から能動的意思決定への移行は、エージェント型LLMフレームワークにより加速している。ReAct や Reflexion などの汎用エージェントアーキテクチャを踏まえ、ポートフォリオ管理や取引シミュレーションのためのエージェント型ループが提案されている。FinMem のように、短期的作業記憶と長期的エピソディック記憶を統合して、ニュース配信に応じて意思決定を動的に調整するシステムもある。また StockAgent のように、複数エージェントの枠組みを用いて市場ミクロ構造をシミュレーションし、自律的なトレーダー、分析者、マーケットメーカー間の相互作用をモデル化する事例もある。これらのエージェント型ループには外部ツール（検索エンジン、計算機、データベース）が組み込まれることが多いが、推論は未構造のテキスト信号や単純な価格時系列に限られ、複雑な企業財務諸表を跨ぐ深層的・多段階の構造推論を実行する能力には欠ける。長期的な株式評価や利益予測には、これらの能力が前提となる。

#### 2.3 構造化財務推論とセンチメント主導のアルファの比較
センチメント主導の取引システムは、市場センチメントが短期的な価格動向の先行指標であるという仮定の下で機能することが多い。しかし、センチメントは一時的な受注・流動性動態や行動バイアスを捉え得る一方、ノイズが大きく信号対ノイズ比が低く、根拠となる実体経済の現実から乖離するリスクがある。対照的に、古典的な定量ファイナンスは、財務諸表（貸借対照表、損益計算書、キャッシュフロー計算書）といった企業の中核財務情報を検討するファンダメンタル分析に基づき、誤評価されている有価証券を同定する。AAARTS は、非構造的テキスト推論と構造化定量モデリングの間のギャップを埋めるものであり、売上認識（P/L）、運転資本の変動（BS）、事業キャッシュ創出（CF）といった三大財務諸表間の相互依存関係を考慮したエージェントループを実装する。これにより、センチメント単独では検出できない、業績品質の異常（例: 純利益と営業キャッシュフローの乖離）を検知できるため、構造的な財務メカニクスに基づく高説明性・低減衰のアルファ信号を生み出すことが可能となる。

### 3. 手法: AAARTS のエージェント型アーキテクチャ

*   System Design Philosophy: 単なるLLMラッパーではなく、複数のエージェントが協調する自律型ループとして設計する。
*   Key Components:
    1.  Hypothesis Generator（LES エージェント）: 財務データとニュースから「なぜこの企業は成長するのか」という仮説を生成する。
    2.  Verification Pillar（The Oracle）: EDINET-Bench 等の客観的基準を用いて、生成された仮説を厳密に評価し、定量的なスコアを算出する。
    3.  Loop Supervisor（Decision Gate）: 検証結果が基準（例：Precision > 70%）を満たさない場合、仮説をピボット（修正）させ、自己改善ループを回す。
*   Data Ingestion: EDINET XBRL からの summary、bs、pl、cf の抽出。
*   Agentic Reasoning: gpt-4o を用い、財務諸表間の「数値の矛盾（例：売上減＋純利増）」を論理的に説明する連鎖推論（Chain-of-Thought）の実装。

### 4. 実験設定

*   Dataset: SakanaAI/EDINET-Bench の earnings_forecast テストセット。
*   Baselines:
    - Naive Prediction（常にマジョリティクラスを予測）。
    - 従来の機械学習モデル（XGBoost + 財務指標）。
    - 素のLLM（Zero-shot prompt）。
*   AAARTS Configuration: システムの推論エンジン（src/commands/bench_eval.py の実装）の設定詳細。

### 5. 結果と考察

*   定量評価:
    - AAARTS の Accuracy（例: 55.00%）、Precision（例: 72.73%）、Recall（例: 57.14%）を報告する。
    - ポイント: Accuracy は相対的に低い水準にあるものの、Precision が高い点（72.73%）に注目される。すなわち「システムが増益を増益と判断したときの確度」が投資戦略上高い優位性を有することを示唆する。
*   定性評価・エラー分析:
    - True Positives（成功例）: AAARTS がどのテキスト表現から増益を見抜いたかを分析する（例：先行投資の回収フェーズへの移行、特定セグメントの好調な記述）。
    - False Negatives/False Positives（失敗例）: どの要因で誤予測となったかを検討（例：突発的なマクロ要因、為替変動など、過去の有価証券報告書のテキストだけでは捉えきれない外部ショック）。
*   アブレーション研究:
    - 要約のみ（数値情報）を用いた場合と、テキスト情報（MD&A 等）も併用した場合の精度の違いを検証。AAARTS がテキストから追加のアルファを抽出できることを示す。

### 6. 議論と今後の展望

*   単年度の有価証券報告書だけでなく、過去数年分のテキストの変化（Delta）を捉えることで、Accuracy のさらなる向上が見込まれる。
*   マクロ経済指標（金利、為替）とのマルチモーダル統合の検討。
*   AAARTS が実際のトレーディングシステム（Polymarket や日本株アルゴ）に与える影響の評価。

### 7. 結論

AAARTS は、複雑な日本語財務文書から自律的に業績の方向性を高精度で予測できることを示した。定性テキストは、クオンツ投資における次世代のアルファ源泉となり得る。
