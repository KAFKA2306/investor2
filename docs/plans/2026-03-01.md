# 2026-03-01 開発日記：非伝統的データから直交アルファを自律抽出—先行指標（Lead-Lag）への大冒険である

**タイトル**: 2026-03-01 開発日記：非伝統的データから直交アルファを自律抽出する試み—先行指標（Lead-Lag）への新たな展開  
**目的**: EDINETおよび e-Stat などの非伝統的データ（オルタナティブデータ）を用いた自律型アルファ生成パイプラインの進捗と、システム全体の機能拡張を記録・共有する。  
**解決したい課題**: アルファの減衰（有効性の低下）、情報のタイムラグ、上位シグナル母数の不足を解消し、他者が模倣できない「直交アルファ」を継続的に発見する。

### 追記：実証の深化と統計的堅牢性 (09:15)

以下の観点で、アルファの強化と検証の堅牢性を高めた。

1.  IC（インフォメーション・コーリレーション）の安定化

    有価証券報告書（10-K）の公表後、市場が認識するまでに遅延が生じるケースがある。このラグをアルファとして捕捉するため、過去の価格データと統合して検証を進めている。

## 追記：儀式を卒業！魂の「鉄壁検問所（Strict Validation）」実装で真の堅牢性を手に入れた

型チェックを行ったつもりで実体をおざなりにしない、という指摘を受けて、重大な修正を実施した。

### 1) 爆弾撤去：`await` 忘れという不具合の修正
`memory.getEvents(20)` を待機させずに Record 配列として扱おうとする誤りを修正した。型キャストのみで誤魔化すことなく、適切に待機して実体を取得する処理に置換した。

### 2) TODO scaffold：`_` 接頭辞による未使用の明示
将来の拡張を想定した変数（sharpe, ic など）に対し、未使用を明示するために `_` 接頭辞を用いる方針を採用した。

### 3) JSON の検問強化
未知のデータを含む場合の安全性を高めるため、`JSON.parse` の検証を強化し、型と実体の整合性を runtime で確認する。

### 4) 結論
誠実性は型に宿り、実体は await に宿る。今後も「意味のある堅牢性」を追求する。

## 追記：嘘を許さない！魂の「監査UI（Evidence Ledger）」への再設計、完了

フロントエンドを「単なる表示画面」から「真実を暴くための監査機能を有するシステム」へ再設計した。

### 1) UI が監査表になる：`MetricCard` の機能
- **sourcePath**: 表示値の出典ファイル・項目を辿れるようにした。  
- **Mismatch! 警告**: 表示データと生データにズレがあれば赤字で警告する。  
- **再計算 (Recompute)**: フロントエンド側で Sharpe や MaxDD を算出し、バックエンドの値と照合するダブルチェック機能を実装した。

### 2) 証拠の三種の神器：3指紋 SourceBadge
- Code：git commit hash（コードの指紋）  
- Data：ideaHash（データの指紋）  
- Run：runId と実行環境（実行の指紋）  
この3つを揃えることで再現性の高い証拠となる。

### 3) 誠実な Rolling IC：PROXY の明示
データが不足して前日リターンを代理として使用した場合には、 PROXY の透かしを表示する。わかっていることと仮定していることを混ぜない。

### 4) 研究者のための新兵器：Cross Section & 比較モード
- 散布図：特定日の予測因子と実際リターンを全銘柄で一度に比較可能にした。  
- 研究ログ比較：過去のアルファ候補を並べ、要因の良否を数値で比較・検討できる。

### 実装上の方針
- TypeScript の型定義を強化：`SourcePath` や `DerivationSpec` など、監査用途の型を厳密に定義した。  
- フロントエンドは、結果の提示だけでなく自己検証を継続する設計となっている。

## 追記：アルファ探索の「魂の進化」—— 4つの根本問題を解消し、本質的価値を獲得

ただループを回すだけでは学習には不十分である。以下の4つの根本問題を修正し、強化学習ループの本質的価値を高めた。

| 問題 | 修正前 | 修正後 | 金融・知能的意義 |
|---|---|---|---|
| Novelty Score | 全候補に 0.9 固定 | Jaccard 類似度で適切に算出 | 同一性の高い因子の反復試行を抑制し、多様な戦略を探究 |
| スコアの時点ずれ | LLMの予測をバックテスト前に使用 | 後追い計算で評価 | 実際の収益性で評価を行い、後出しの影響を排除 |
| ACEフィードバック死亡 | addBullet() 忘れで記憶ゼロ | アイデア保存時に bullet を作成 | 過去の勝ち筋・負け筋を記憶に蓄積し、LLMが賢く動くよう支援 |
| 学習の可視化ゼロ | 成果物があるかのみ | CycleSummary JSON を出力 | サイクルごとの成長を定量的に追跡できる |

### インサイト（内緒の気づき）

1. ACEメモリは「金融特化のRAG」であり、helpful_count の高い bullet を優先して提示する仕組みは再学習を伴わず知識を更新する。  
2. Jaccard novelty は「因子 Zoo」への処方箋となり、類似因子を除外することで真に新しい価値を追求できる。
