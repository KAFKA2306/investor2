# ADR-001: ハーネスエンジニアリング、はじめちゃうよっ！ ✨

## ステータス： 承認済みっ！ (Accepted) 💖

## 日付： 2026-03-09 📅

## はいけい (Context) 🌸

2026年3月現在、Claude CodeやCodexみたいなエージェントちゃんたちがすごーく賢くなったから、人間さん向けの説明文よりも、エージェントちゃんが「これ読んでっ！」ってわかるポインタや、自動で動くツールがとっても大事になったんだよっ！長いREADMEとかはすぐ古くなっちゃう（Context Rot）から、エージェントちゃんが困っちゃわないようにしたいのっ！

## きめたこと (Decision) 🎀

このリポジトリでは「ハーネスエンジニアリング」を最強のルールにするよっ！

1. **動くものが正義っ！**: テストや型定義、リンターの設定が一番の真実だよっ。
2. **ポインタは短くっ！**: `CLAUDE.md` と `AGENTS.md` は50行以内にして、エージェントちゃんが迷わないようにするよっ！
3. **ガードレール設置っ！**: 大事な決定は「ADR」に残して、リンターで「これダメだよっ！」って言わせるようにするのっ。
4. **自動でキレイキレイっ！**: Claude CodeのHookを使って、ファイルを保存するたびに自動でリンターちゃんを走らせるよっ！

## どうなるの？ (Consequences) 🍭

- `README.md` がすっごくスリムになって、見やすくなるよっ！
- ファイルを保存したとき、ちょっとだけ待つかもしれないけど、そのぶんコードがいつもピカピカになるよっ！
- 新しい決定は、必ずADRに可愛く残していくよっ！
