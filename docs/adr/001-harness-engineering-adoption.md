# ADR-001: ハーネスエンジニアリングの導入を開始する

## ステータス：承認済み（Accepted）

## 日付： 2026-03-09

## 背景 (Context)

2026年時点で、Claude Code や Codex のようなエージェントは高度に進化しており、人間向けの説明文よりも、エージェントが「これを読め」と理解できる指示や自動で実行するツールの重要性が高まっている。長大な README は迅速に陳腐化する（Context Rot）ため、エージェントが混乱しないよう整備を推進する必要がある。

## 決定事項

このリポジトリでは「ハーネスエンジニアリング」を最重要のルールと位置づける。

1. 動作することを最優先とする。テスト、型定義、およびリンターの設定を最重要とする。
2. 指示は簡潔にする。`CLAUDE.md` および `AGENTS.md` は50行以内に収め、エージェントが混乱しないようにする。
3. ガードレールを設置する。重要な決定は ADR に記録し、リンターが適切に警告を出すようにする。
4. 自動整合性の維持を実行する。Claude Code のフックを用い、ファイル保存時に自動でリンターを実行する。

## どうなるのか？ (Consequences)

- `README.md` が大幅に簡素化され、閲覧性が向上する。
- ファイルを保存した際には若干の遅延が生じることがあるが、その分コード品質が常に向上する。
- 新たな決定は必ず ADR に記録する。
