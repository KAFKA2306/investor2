# ADR-002: 全自律型メンテナンスループの導入

## ステータス： 提案中

## 日付： 2026-03-09

## 背景 (Context)

リポジトリの衛生状態（ドキュメントの腐敗、リンター違反、アーキテクチャの逸脱）を人間が継続的に監視することは困難である。ハーネスエンジニアリングをさらに進化させ、エージェント自身が「ここは不適切だ」と認識し、自己修復する仕組みを構築したい。

## 決定事項

1. **リンターを「先生」にする**:
   - リンターのエラーメッセージを単なる指摘にとどめず、修正手順を具体的に提示する。
   - 例: 「DTOは `src/dtos/` に配置せよ。現在は `src/domain/` にあるため、移動を実行せよ。」

2. **Hookで「即時修正」ループ**:
   - Claude Code の `PostToolUse` Hook を用い、ファイルを書き換えた直後に `lint --fix` を自動実行する。
   - 修正不能なエラーは、モデルのコンテキストへ自動で注入され、次回の実行時に修正を試みさせる。

3. **「保守エージェント」の定期巡回**:
   - `Taskfile` に `task maintenance:gc`（ガベージコレクション）を追加する。
   - これは、未使用ファイル、古くなったドキュメント、過度に複雑なコードを検出し、自動的にリファクタリングPRを作成する業務である。

## 影響 (Consequences)

- 人手によるコードレビューでのフォーマット指摘が不要になる。
- リポジトリが自動的に整理され、整然とした状態を維持できる環境が構築される。
- エージェントはエラーメッセージを学習しつつ、業務遂行能力が向上する。
