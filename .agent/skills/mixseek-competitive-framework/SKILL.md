# Mixseek Competitive Framework for TypeScript

## Overview

This skill orchestrates a competitive evaluation of multiple Qlib signal formula candidates generated by LesAgent. Each candidate is backtested independently, ranked by Sharpe ratio, and the winner is selected for downstream CqoAgent evaluation.

**When to use:**
- LesAgent has generated 2+ Qlib formula candidates
- You need deterministic ranking by performance (Sharpe ratio)
- Output feeds directly into CqoAgent's standardized evaluation pipeline
- Goal: select the single best candidate formula before detailed statistical testing

## Input Specification

```json
{
  "candidates": [
    {
      "factor_id": "string",
      "formula": "Qlib expression string",
      "economic_mechanism": "string (optional description)"
    },
    // ... more candidates
  ],
  "backtest_config": {
    "start_date": "YYYY-MM-DD",
    "end_date": "YYYY-MM-DD",
    "universe": "jp_stocks_300",
    "rebalance_frequency": "daily"
  }
}
```

## Evaluation Pipeline

### Step 1: Prepare Candidates
- Validate each formula as syntactically correct Qlib expression
- Extract factor_id and formula for tracking
- Log candidate count and mechanism descriptions

### Step 2: Parallel Backtest Execution
For each candidate:
1. Execute Qlib backtest over the specified date range
2. Compute daily rank correlation between signal values and next-day returns
3. Calculate Sharpe ratio from daily correlations: `Sharpe = mean(correlations) / std(correlations)`
4. Capture Sharpe, IC (information coefficient), max drawdown
5. Store results with factor_id for tracking

### Step 3: Ranking & Winner Selection
1. Sort candidates by Sharpe ratio (descending)
2. Identify the winner (highest Sharpe)
3. Record ranking with all candidates for transparency
4. Calculate performance delta between winner and second place

### Step 4: Output for CqoAgent

```json
{
  "winner": {
    "factor_id": "string",
    "formula": "Qlib expression",
    "economic_mechanism": "string",
    "performance": {
      "sharpe": 2.15,
      "ic": 0.0424,
      "max_drawdown": 0.128
    }
  },
  "rankings": [
    {
      "rank": 1,
      "factor_id": "...",
      "sharpe": 2.15
    }
    // ... all candidates ranked
  ],
  "competition_metadata": {
    "total_candidates": 3,
    "evaluation_date_range": "YYYY-MM-DD to YYYY-MM-DD",
    "ranking_metric": "sharpe_ratio"
  }
}
```

## Implementation Notes

- **Deterministic**: Results depend only on formula and data, not random seed
- **Transparent**: All candidates ranked; user can inspect runner-up
- **CqoAgent-compatible**: Output format integrates directly into existing quality-gate workflow
- **Performance**: Parallel backtest execution over all candidates; final ranking takes seconds

## Edge Cases

- **Tie (same Sharpe)**: Select candidate by factor_id alphabetically
- **Invalid formula**: Log error, exclude candidate from ranking, continue with others
- **Single candidate**: Still output as winner with empty rankings array
- **No valid candidates**: Return error with diagnostics

## Integration with Project Pipeline

This skill operates at the competitive evaluation stage, positioned between LesAgent (candidate generation) and CqoAgent (quality audit):

```
LesAgent (generate 2+ Qlib formulas)
    ↓
[mixseek-competitive-framework] ← YOU ARE HERE
    ↓ (winner formula + performance)
CqoAgent (AAARTS audit: Sharpe, IC, constraints)
    ↓
Pipeline decision (GO / HOLD / PIVOT)
```

The winner's Sharpe and IC flow into CqoAgent's thresholds (Sharpe > 1.8, IC > 0.04).
